二、 細項類別 (Sub-Category) 提取與分流定義

數據提取時，必須依據特定表單之版面結構，將檔案路由至正確的細項欄位中：

  • QC10002-R02 (原物料品檢)：
    • 優先依據檔案所在之相對路徑 (relPath) 進行物料關鍵字對照，例如：B膠 歸於 物料-B膠，紙箱 歸於 物料-紙箱。
    • 若包含 射出D 關鍵字，則歸於 射出D 欄位。
    • 若路徑包含 原料 且無特定匹配，或無任何匹配，則預設歸為 原料。
  • QC10006-R02 (半成品)：
    • 標準路徑：細項名稱取自檔案所在之首層子目錄名稱，並剝除年份後綴（如 Vivus-2025 → Vivus）。
    • 關鍵字驅動路由：若檔案名稱或相對於 QC 根目錄的路徑包含 半成品品檢表（例如 零組件入庫-2023/半成品品檢表2023(限組件用)/），強制路由至 QC10006-R02 且細項類別固定為 裝配C。
    • 月份提取彈性化（支援 2015-2099）：
      • 策略 1: Sheet name 字母模式 (\d{2})([A-L]) + 年份驗證 (15-99)，如 PJW25D13 → D=4月
      • 策略 2: 檔名模式 半成品品檢表\d{4}[-_]\d{1,2}，如 半成品品檢表2023-01.xlsx → 1月
      • 策略 3: Sheet name YYMMDD 模式 (\d{2})(\d{2})(\d{2})，如 230115 → 1月
  • QC10007-R03 (零組件入庫)：
    • 細項名稱為首層子目錄名，但嚴格限定並分流至：射出、射出A、射出C、射出D、射出D(組件)、Tubing、裝配A、裝配B、裝配C 欄位。
    • 若首層子目錄名為 射出D 且相對路徑中無「組件」標記，系統會將該檔案重定向至 QC10002-R02 的 射出D 中統計。
  • QC10008-R02 (出貨檢驗)：
    • 工作表名稱包含 ICU 歸於 ICU 欄位，其餘歸入 其他。

三、 品檢月份 (Month) 提取演算法優先級

由於各品檢 Excel 檔案中日期填寫位置可能因版本修訂而偏移，系統採用以下 5 層降級（Fallback）優先順序進行月份提取：

【優先級 1】特定工作表儲存格精準讀取 (Cell Exact Value) ├─ QC10002-R02: 優先讀取 N4 (Rev C)；無效則讀取 O4 (Rev D)。 ├─ QC10006-R02 / QC10007-R01: 讀取 N5。 ├─ QC10007-R03: 讀取 O4。 ├─ QC10008-R02: 讀取 R6。 └─ QIP (QC10004-R02): 讀取 Q4 (提取 YYMMDD 中的 MM)。 ▼ 失敗 【優先級 2】輕量化工作表首 15 列自動日期掃描 (Header Range Date Scan) └─ 遍歷 Column A 至 Z 的前 15 列單元格，匹配任意年份的日期值或字串，提取其月份。 ▼ 失敗 【優先級 3】檔案名稱正規表示式匹配 (Filename Regex Match) ├─ 模式 A: 檔名結尾為 `YYYY-MM.xlsx` (如 `2025-02.xlsx` → 2月) ├─ 模式 B: 檔名內嵌 `YYYYMMDD` (如 `20250312...xlsx` → 3月) └─ 模式 C: 檔名結尾為簡寫 `YYMM.xlsx` 或 `-M.xlsx` (如 `2504.xlsx` → 4月) ▼ 失敗 【優先級 4】工作表名稱或資料夾路徑特徵 (Sheet/Folder Name Match) ├─ 工作表名稱包含 `MM月` (如 `12月` → 12月) └─ 相對路徑包含 `MM月` 或 `Tubing-YYYY-MM` (如 `Tubing-2025-06` → 6月) ▼ 失敗 【優先級 5】後綴月份字母對照 (Month Suffix Letter - A=1, B=2 ... L=12) ├─ 適用於零組件入庫 (QC10007-R03): 檔名後綴為大寫字母 (如 `裝配B-2025A.xlsx` → 1月) └─ 適用於半成品 (QC10006-R02): 工作表名稱含有 `YYLetter` (如 `25D` → 4月) 【特殊路由：關鍵字驅動半成品品檢表】 └─ 檔名/路徑包含「半成品品檢表」→ 強制 QC10006-R02 / 裝配C ├─ 策略 1: Sheet name `(\d{2})([A-L])` + 年份驗證 15-99 (PJW25D13 → 4月) ├─ 策略 2: 檔名 `半成品品檢表\d{4}[-_]\d{1,2}` (半成品品檢表2023-01 → 1月) └─ 策略 3: Sheet name `(\d{2})(\d{2})(\d{2})` (230115 → 1月)

四、 空白預建樣板過濾守衛 (Blank Guard)

針對零組件入庫品檢 QC10007-R03 表單，專案在 RawData 中通常會預建全年度 1-12 月的空白模板檔案。為防止未檢驗月份產生幻象數據，本專案在前後端均實作了空白模板守衛：

過濾核心邏輯：
在處理 `QC10007-R03` 類型工作表時，代碼會檢索工作表的 Cell G4（對應 JSON 轉出矩陣的 json[3][6]，即「批號」欄位）：
  • 如果該儲存格的值為 '' (空字串)、null、undefined、數值 0 或字串 "0"，則直接判定為空白樣板。
  • 一體適用性：此過濾邏輯不限於特定月份，而是對 1 月至 12 月的所有工作表均進行一致性攔截，直接跳過該工作表的數據統計。

五、 特定表單特殊邏輯 (Deduplication & QIP)

1. QIP 尺寸製程檢驗 (QC10004-R02) 獨立統計規則

依照製程品檢規格定義，QIP 表單不進行常规的單元格掃描，而是直接針對 射出檢驗 與 押出檢驗 目錄做特殊分流：

統計項目 資料夾限制範圍 Setup (架機數) 計算規則 Patrol (巡檢數) 計算規則
QIP 射出 (Injection) RawData/{year}/射出檢驗-{year}/QIP-{year}(1~10)/ 及其子目錄 該月份資料夾內所有的 .xlsx 檔案數（排除 ~$ 開頭的暫存檔）。 1. 僅保留工作表名稱符合 Date Code 格式（即 6 位數字或 6 位數字加一個字母，正則 /^\d{6}[a-zA-Z]?$/）的工作表。
2. 去重規則：單一檔案內後綴去重（例如 260521-1 與 260521(2) 歸併為 260521），去重後的工作表數量即為巡檢數。
QIP 押出 (Extrusion) RawData/{year}/押出檢驗-{year}/ 及其子目錄 符合 6 位數 Date Code 檔名的檔案個數（一個檔案計為 1 次 Setup）。 1. 排除工作表名稱中包含 SETUP 的工作表。
2. 僅統計工作表名稱符合 Date Code 格式且於單一檔案內去重後的工作表數量。

2. 完成品裝配品檢 (QC10007-R01) 批次去重邏輯

同一批次完成品出廠時，其品檢表內可能包含多個續頁工作表（例如 251201 (1), 251201 (2), 251201 (NG) 等）。

  • 去重做法：代碼會剔除工作表尾部的括號及流水號（正則 /\s*\([^)]+\)\s*$/），還原為基礎批次名稱（如 251201）。
  • 統計約束：在同一個 Excel 檔案中，同一個基礎批次名稱僅會被統計一次，避免因多頁檢驗報告導致合格批次數量虛增。

為確保系統在不同運行環境下產生一致的 Excel 報表，此處列出兩套 Pipeline 的主要對齊關係：

目標年份解析 透過 CLI 參數傳入 (node etl_pipeline.cjs 2027)。 智慧偵測上傳路徑與檔名特徵，自動設定並提供下拉選單 override。 完全對齊

七、 關鍵字驅動路由與彈性月份提取 (v3.2 新增)

為解決 2023 年及以前資料夾結構與 2025/2026 年不同的問題，系統引入關鍵字驅動路由機制：

1. 問題背景

年份 半成品品檢表路徑 問題
2025/2026 RawData/2025/裝配檢驗-2025/BD/ 路徑包含「裝配檢驗」→ 正常匹配 QC10006-R02
2023 RawData/2023報表/零組件入庫-2023/半成品品檢表2023(限組件用)/ 路徑包含「零組件入庫」→ 被誤判為 QC10007-R03

2. 解決方案：關鍵字驅動路由

  • 檔名/路徑關鍵字優先：若檔案名稱或相對於 QC 根目錄的路徑包含 半成品品檢表，強制路由至 QC10006-R02，子類別固定為 裝配C。
  • 資料夾映射優先級：在 QC_FOLDER_MAP 中，半成品品檢表 排在 零組件入庫 之前，確保優先匹配。

3. 彈性月份提取 (支援 2015-2099)

針對關鍵字驅動的半成品品檢表檔案，系統採用三層彈性月份提取策略：

策略 正則模式 範例 提取月份
1. Sheet name 字母模式 (\d{2})([A-L]) + 年份驗證 (15-99) PJW25D13 D → 4月
2. 檔名月份模式 半成品品檢表\d{4}[-_]\d{1,2} 半成品品檢表2023-01.xlsx 01 → 1月
3. Sheet name YYMMDD 模式 (\d{2})(\d{2})(\d{2}) 230115 01 → 1月

4. 驗證結果

已建立自動化驗證腳本 scratch/test_semi_finished_month.cjs，測試 2015-2027 年共 468 個用例，100% 通過。
