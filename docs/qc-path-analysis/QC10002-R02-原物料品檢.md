**實現函數**：[`detectQCFromFolder()`](file:///d:/Self-developed_Apps/FileName_WorkSheet_Extract/src/utils/browserETL.js#L36-L44)

```javascript
function detectQCFromFolder(dirname) {
  const m = dirname.match(/QC\d{5}-R\d{2}/i);
  if (m) return m[0].toUpperCase();
  const keys = Object.keys(FOLDER_QC_MAP);
  for (let i = 0; i < keys.length; i++) {
    if (dirname.indexOf(keys[i]) >= 0) return FOLDER_QC_MAP[keys[i]];
  }
  return null;
}
```

### 3.2 映射邏輯

```javascript
function getRawSubCategory(qc, relPath, fileName, sheetName, qcFolder) {
  if (qc === 'QC10002-R02') {
    const parts = relPath.split('/');
    const p0 = parts[0].replace(/[-_]\d{4}$/, '').replace(/\s+/g, '');
    
    if (p0 === '原料') return '原料';
    
    if (p0 === '物料') {
      if (parts.length > 1) {
        const sub = parts[1].replace(/[-_]\d{4}$/, '').replace(/\s+/g, '');
        if (sub) return '物料-' + sub;
        return '物料-' + parts[1];
      }
      let name = fileName.replace(/\.xlsx$/i, '');
      name = name.replace(/[-_]\d{4}[-_]\d{1,2}$/, '');
      name = name.replace(/[-_]\d{4}$/, '');
      name = name.replace(/[-_]\d{1,2}$/, '');
      name = name.replace(/\s+/g, '');
      return '物料-' + name;
    }
    
    if (parts[0].indexOf('射出D') >= 0) return '射出D';
    return null; // 映射失敗
  }
  // ... 其他 QC 標籤處理
}
```

### 3.4 路徑解析細節

**年份後綴移除**：
- 正規表達式：`/[-_]\d{4}$/`

- 範例：`物料-B膠_2026` → `物料-B膠`

**空白字元移除**：
- 正規表達式：`/\s+/g`
- 範例：`原 料` → `原料`

## 五、第四層：月份提取邏輯

### 5.1 優先級最高的策略

**實現函數**：[`findDateInSheet()`](file:///d:/Self-developed_Apps/FileName_WorkSheet_Extract/src/utils/browserETL.js#L76-L118)

**QC10002-R02 專用邏輯**：
```javascript
case 'QC10002-R02': // 原物料進料品檢
  cellInfo = getCellValAndFormatted('N4');
  dateInfo = parseDateFromValue(cellInfo.val, cellInfo.formatted);
  if (!dateInfo) {
    cellInfo = getCellValAndFormatted('O4');
    dateInfo = parseDateFromValue(cellInfo.val, cellInfo.formatted);
  }

```

**讀取順序**：
1. 儲存格 `N4` → 解析日期
2. 若失敗 → 儲存格 `O4` → 解析日期

**日期解析支援格式**：
- JavaScript Date 物件
- Excel 日期序列號（40000-50000 範圍）
- 字串格式：`2025/01/15`、`25/01/15`、`250115`

### 5.2 備援策略集合

**實現函數**：[`extractRawMonth()`](file:///d:/Self-developed_Apps/FileName_WorkSheet_Extract/src/utils/browserETL.js#L275-L351)

| 策略編號 | 來源 | 正規表達式 | 範例 |
|---------|------|-----------|------|
| 1 | 檔案名稱 | `(\d{4})[-_](\d{1,2})\.xlsx` | `xxx-2025-01.xlsx` → 1 月 |
| 2 | 檔案名稱 | `(\d{4})(\d{2})\d{2}` | `xxx20250115.xlsx` → 01 月 |
| 3 | 檔案名稱 | `(\d{2})(\d{2})\d{2}` | `xxx250115.xlsx` → 01 月 |
| 4 | 檔案名稱 | `(\d{2})(\d{2})\.xlsx` | `xxx2501.xlsx` → 01 月 |
| 5 | 檔案名稱 | `[-_](\d{1,2})\.xlsx` | `xxx-1.xlsx` → 1 月 |
| 6 | 相對路徑 | `[-_](\d{1,2})$` | `原料-1` → 1 月 |
| 7 | 相對路徑 | `(\d{1,2})月` | `原料-1月` → 1 月 |
| 8 | Sheet 名稱 | `(\d{2})(\d{2})\d{2}` | Sheet `250115` → 01 月 |
| 9 | Sheet 名稱 | `(\d{1,2})月` | Sheet `1月` → 1 月 |
| 10 | 內容掃描 | `findDateInSheetFallback()` | 前 20 列搜尋日期格式 |

### 5.3 特殊處理：跨年情況

## 七、潛在風險點分析

### 7.1 風險 R1：子分類匹配過於嚴格

**描述**：
- `getRawSubCategory` 要求路徑必須完全匹配 `原料`、`物料-xxx` 等精確字串
- 使用 `===` 進行比較，不支援模糊匹配

**影響**：
- 若使用者建立 `原材料`（多了一個「材」字），會返回 `null` 導致檔案被跳過
- 若路徑包含額外的斜線或空白，可能匹配失敗

**嚴重性**：中高  
**建議**：在 `getRawSubCategory` 中添加容錯機制，允許近似匹配或使用 `includes()` 而非 `===`

---

### 7.2 風險 R2：物料子分類命名不一致

**描述**：
- `rawSubCatMap` 中處理了 `物料-塑膠袋40*50`（星號）與 `物料-塑膠袋40X50`（X）
- 但在 `getRawSubCategory` 中提取的路徑可能產生其他變體

**影響**：
- 若使用者使用 `物料-塑膠袋40x50`（小寫 x），可能無法正確映射到索引 7
- 導致數據分散到不同的欄位

**嚴重性**：中  
**建議**：
1. 統一命名規範，強制轉換為大寫 X
2. 在 `rawSubCatMap` 中添加更多備援匹配

---

### 7.3 風險 R3：射出D 的特殊處理衝突

**描述**：
- 在 `getRawSubCategory` 中，路徑包含「射出D」時返回 `射出D` 子分類（歸類為 QC10002-R02）
- 但在 `runETLInBrowser` 的 QC10007-R03 處理區塊（第 484-486 行），若路徑包含 `射出D` 但不包含 `射出D(組件)`，會將 actualQC 覆蓋為 `QC10002-R02`

**影響**：
- 可能造成類別混淆，同一個檔案被錯誤分類
- 需要確認業務邏輯是否正確

**嚴重性**：高  
**建議**：審查 QC10007-R03 的覆蓋邏輯，確保不會錯誤覆蓋 QC10002-R02 的映射

---

### 7.4 風險 R4：月份提取依賴多個備援策略

**描述**：
- 若儲存格 N4/O4 無日期，會依序嘗試 9 種備援策略
- 不同策略可能提取出不同月份

**影響**：
- 若檔案名稱包含日期（如 `20250115`），同時路徑也包含月份（如 `原料-1`），可能產生不一致
- 優先順序雖已定義，但若邏輯變更可能導致回溯行為

**嚴重性**：中  
**建議**：
1. 添加日誌記錄，記錄實際使用的策略編號
2. 在 DEV_LOG.md 中追蹤月份提取失敗的案例

---

### 7.5 風險 R5：FOLDER_QC_MAP 匹配順序

**描述**：
- `detectQCFromFolder()` 使用 `Object.keys(FOLDER_QC_MAP)` 遍歷匹配
- JavaScript 物件的鍵順序在 ES2015+ 中是有定義的，但依賴字面順序可能有風險

**影響**：
- 若未來新增的關鍵字與現有關鍵字有包含關係（如「原料」與「原材料」），可能產生歧義

**嚴重性**：低  
**建議**：考慮改為陣列順序，或添加更明確的優先級標記
